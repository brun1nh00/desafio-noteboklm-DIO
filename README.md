# Engenharia de Prompts:
         
Prompt 1: Explique a diferença entre Analista de Negócios e Analista de Dados utilizando exemplos práticos de empresas de tecnologia.

Embora as funções de Analista de Negócios e Analista de Dados muitas vezes se sobreponham, a principal diferença reside no foco: enquanto o analista de dados se concentra em extrair insights técnicos e padrões de conjuntos de dados brutos, o analista de negócios utiliza esses insights para definir estratégias e guiar a tomada de decisões corporativas
De forma simplificada, o analista de dados produz a evidência ("o quê" e "por quê"), enquanto o analista de negócios fornece a direção ("e agora?")
Aqui estão as principais diferenças ilustradas com exemplos de gigantes da tecnologia:

1. Foco e Objetivos
Analista de Dados: Busca a descoberta de conhecimento e padrões ocultos em dados granulares e muitas vezes não estruturados
No Google, por exemplo, um analista de dados pode analisar o desempenho de algoritmos de segmentação de anúncios automatizados para entender quais variáveis influenciam os cliques

Analista de Negócios: Foca na execução acionável e na criação de valor corporativo
Utilizando os dados de anúncios do Google, o analista de negócios recomendaria uma reestruturação das equipes de vendas e suporte para alinhar as operações ao modelo "AI-first" da empresa

2. Habilidades e Ferramentas
Analista de Dados: Requer proficiência técnica em linguagens de programação como Python e R, domínio de SQL para manipulação de bancos de dados e ferramentas de visualização como Tableau ou Power BI
Eles lidam intensamente com a limpeza e normalização de dados "sujos" para torná-los utilizáveis

Analista de Negócios: Embora use ferramentas como Excel e SQL, seu conjunto de habilidades enfatiza o pensamento crítico, a comunicação e o entendimento de objetivos organizacionais
No caso da Snowflake, analistas de negócios usaram métricas de uso interno para planejar a transição estratégica de um armazém de dados em nuvem para uma plataforma de nuvem de dados focada em IA

3. Exemplos Práticos em Empresas de Tecnologia
Empresa
Ação do Analista de Dados
Ação do Analista de Negócios
Microsoft
Analisou metadados de calendários de 1.200 engenheiros para medir tempos de deslocamento entre prédios

Hipotetizou que a densidade espacial aumentaria a colaboração e calculou uma economia anual de US$ 520.000 em horas produtivas

E-commerce (ex: Amazon)
Utiliza algoritmos de clustering para segmentar clientes com base em comportamentos de navegação e compras passadas

Traduz esses segmentos em prioridades de campanha de marketing e decide a alocação de orçamento para cada canal

Tech/Banking
Realiza a mineração de dados para identificar padrões de comportamento que indicam possíveis fraudes em transações

Define as regras de negócio e políticas que o sistema deve seguir para bloquear transações suspeitas sem prejudicar a experiência do cliente

4. O Papel da Colaboração
Em ambientes de tecnologia modernos, esses papéis são complementares
O analista de dados funciona como o "cartógrafo", criando o mapa através da interpretação dos dados, enquanto o analista de negócios age como o "navegador", usando esse mapa para traçar o melhor curso para o sucesso da organização
Em empresas menores ou áreas específicas, pode surgir o papel híbrido de Analista de Business Intelligence (BI), que combina a coleta e análise técnica com a aplicação estratégica direta
